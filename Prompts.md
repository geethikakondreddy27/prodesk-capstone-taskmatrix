# AI Prompt Engineering Log

## Project: FlowStack

This document captures the key AI-assisted research prompts used during the planning and architectural design phase of the FlowStack capstone project.

---

# 1. Project Selection

## Prompt

Compare an Enterprise Learning Management System, Healthcare EHR Dashboard, and Agile Project Management platform from the perspectives of software architecture, scalability, implementation complexity, database design, portfolio value, and real-world relevance. Determine which project is most suitable for a five-week Full Stack MERN capstone.

---

# 2. Product Research

## Prompt

Jira and Asana already dominate the Agile project management market. Analyze their strengths and limitations, then identify opportunities to introduce a practical feature or workflow that improves the user experience without deviating from standard Agile practices.

---

# 3. System Architecture

## Prompt

Design a high-level architecture for a Full Stack MERN-based Agile project management platform. Explain how the frontend, backend, database, authentication, and real-time communication components interact with one another.

---

# 4. Database Design

## Prompt

Identify the MongoDB collections required for an enterprise Agile project management platform. Define the relationships between Users, Workspaces, Projects, Tasks, Comments, Notifications, and Activity Logs while keeping the design scalable and maintainable.

---

# 5. Technology Stack Analysis

## Prompt

Analyze suitable technologies for building a commercial-grade project management application. Compare React with Next.js, MongoDB with PostgreSQL, and different authentication approaches before selecting the final technology stack.

---

# 6. Feature Planning

## Prompt

Prepare an implementation roadmap by organizing features into MVP, secondary features, and future enhancements. Prioritize functionality based on business value, development effort, and project timeline.

---

# 7. UI/UX Planning

## Prompt

Determine the essential application screens required for an Agile project management platform. Describe the information hierarchy, navigation flow, and dashboard components that provide the best user experience for administrators, project managers, and team members.

---

# 8. Scalability & Future Scope

## Prompt

Evaluate how the application architecture can support future growth. Consider modular backend development, reusable frontend components, API organization, database optimization, and integration possibilities for future releases.

---

---

# 9. Authentication Workflow Design

## Prompt

Design a secure authentication workflow for a Full Stack MERN application. Explain how user registration, login, password hashing, JWT generation, protected routes, and logout should work together while keeping the architecture modular and production-ready.

---

# 10. Backend Folder Architecture

## Prompt

Recommend a scalable backend folder structure for a MERN application implementing authentication. Justify the responsibilities of controllers, routes, middleware, models, configuration files, and utility modules to improve maintainability as the project grows.

---

# 11. Authentication API Design

## Prompt

Design RESTful authentication endpoints for a project management application. Define the request and response structure for user registration, login, and protected resources while following standard API design principles.

---

# 12. JWT Authorization Strategy

## Prompt

Compare session-based authentication and JWT-based authentication for modern MERN applications. Evaluate their advantages, limitations, scalability, and suitability for a project management platform before selecting the final authentication strategy.

---

# 13. Frontend Authentication Architecture

## Prompt

Design the authentication flow for a React application using React Router. Explain how registration, login, protected routes, logout, and client-side token persistence should interact to provide a secure user experience.

---

# 14. Deployment Planning

## Prompt

Prepare a deployment strategy for a MERN application using Render for the backend and Vercel for the frontend. Identify configuration steps, API integration requirements, CORS considerations, environment variables, and common deployment issues that should be verified before production release.

---

# 15. Authentication Validation

## Prompt

Recommend frontend validation strategies for an authentication system. Evaluate email validation, password confirmation, password strength indicators, and user feedback mechanisms that improve usability without compromising security.

---

# 16. MVP Scope Evaluation

## Prompt

Review the current implementation against the planned product roadmap. Identify which authentication features are essential for the MVP, which project management modules should be deferred to future sprints, and justify the prioritization based on software engineering best practices.
